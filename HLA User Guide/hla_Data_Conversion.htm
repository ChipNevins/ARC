Data Conversion

[15 October 2002]

Overview

Part of the project is to convert the existing Mumps data into Oracle. I am converting the data starting with the most recent data, one month at a time. To convert a particular month, I convert the data for all patients/donors that have a specimen accessioned during that month. The conversion program I use is just the XML writer and XML Reader run in batch mode. There is no separate conversion program except for a small driver.

The data I convert for each patient/donor is:

  • PERSON - ref#, name, DRN, Registration categ, Donor type, Donor location, donor center code, availability, comments
  • SPECIMEN - Number, specimen type, accession date, bleed date, who entered, hospital, comment (Note: inventory will be converted later)
  • HLA TYPE - The full HLA type and its history
  • TESTS - The test code, test type (e.g., HLA Test), order date, result date, result text, who entered, comment.
  • PERSON AUDIT - Converting the Mumps file which audits changes to the person
  • DICTIONARIES - The EMPLOYEE dictionary was converted in full. Deleted entries are preserved as deleted. Other dictionaries have been populated as needed, e.g., when a specimen is converted that was collected at a particular hospital, that hospital is added to the dictionary

Conversion progress
