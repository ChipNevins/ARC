HLA Hot Backup

Last revision: Feb 10, 2005    To print this page, Click here first

 

This document explains the procedure for restoring the Oracle database onto a ‘Hot’ backup database if the HLA1 server is down for an extended period of time.

Procedure for restoring the backup

  1. Get the last good backup tape from the HLA1 server.  Insert this tape into the HLA2 server.  Run the Backup|Restore option, and select ONLY ONE file to restore:  d:\backup\hlaprod.dmp.
  2. Copy file hlaprod.dmp into folder c:\hla\dba\ on the Edmond PC
  3. On Edmond, right-click on the MyComputer icon, select manage, then find the services icon.  Start these services: OracleHLAHOTDB and the Listener.
  4. Start program SQLPlus Worksheet (it’s on the desktop).  Connect as user Hla, password = <hla password>, service = HLAHOTDB.
  5. When you are connected, type in this instruction: ‘start c:\hla\dba\hla_db_clear.sql’.  Wait until this program runs to completion.  Now close the SQLPlus window.
  6. On the desktop, select Start | Programs | Accessories | Command Prompt.  A command window opens, and you see the prompt ‘C>’.  At this prompt, type “c:\hla\dba\import_hlahotdb.bat’.  Wait for this program to finish, then close the command window.
  7. On the Edmond PC,  start the HLA Oracle Application,  Use the login values: username = hla, password = <hla password>, database = hlahotdb.  Verify that you are connected.
  8. Let users onto the system, logging into HLAHOTDB

 

IMPORTANT NOTE:  While you are using the Hot backup, you cannot use the Oracle system on HLA1, even after the hardware is fixed.  Gil or Chip will move the Hot Backup data back to HLA first.
