Employee Setup

Last revision: July 19, 2009    To print this page, Click here first

To set up a new Employee to use Oracle HLA

  1. Create the Oracle account.

    • On the HLA1 server, sign in as administrator

  2. Click Start|Programs|Oracle|Database Administration|Security Manager. The computer opens the Manager Login window

  3. Enter Username: SYSTEM, Password: v*****system, Service: hlaprod. Click OK. The computer opens the Security Manager window. On the left you see a folder named Users.

  4. Right-click the mouse on the 'Users' folder. From the popup menu, select Create. The computer opens the Create User window.

  5. In the General tab, enter the username in the 'Name:' field (no spaces in the name!). Then, enter the password in the 'Enter Password:' field and again in the 'Confirm Password:' field. Remember the username and password you entered!

  6. Click the Role tab. You see a list of roles in the Available section, and you see the CONNECT role in the Granted section. Double click each of these roles: RESOURCE, HLA_USER, LIFC_READER. As you double-click each role, it will be added to the Granted list.

  7. The Granted list should now contain CONNECT, RESOURCE, HLA_USER, LIFC_READER.

  8. Click the Create button. The computer displays the message 'User created successfully'. Click OK to close the message window.

  9. Close the Security Manager window

  10. Enter the EMPLOYEE in HLA Oracle system

    • Log into the HLA Oracle system with your normal password. You must have security level 7 in Hla Oracle to perform this set of steps

  11. From the menu, select Data Entry|Admin|Codes. The computer opens the Codes maintenance window.

  12. Select EMPLOYEE from the list of code types. Using the mouse, right-click in the list of Employe names, then select add.

  13. Enter the Employee name, LAST,FIRST, employee initials

  14. In the lower window, enter the ORACLE NAME and the SECURITY LEVEL. The ORACLE NAME value you enter must be the same as the Oracle username you created above.

  15. These are the security levels: 1-View, 3-Login, 5-ResultEntry, 7-Dictionary maintenance.

  16. Click SAVE. You are done, this employee can now use HLA Oracle

To remove an Employee

  1. Go to window Data Entry|AdminCodes, select the EMPLOYEE code table; then select this Employee. In the bottom window enter Security Level = 0. This will disable HLA Oracle login for this user (the next time they attempt login).

  2. Ask Chip or Gil to delete the Oracle account. Deleting the account does NOT affect the audit report of any activity performed by the user.
