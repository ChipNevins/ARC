Instructions for Heme-Quebec are similar to what you use for other supplier files (like Cryolife or Enterolab):

1.     Carefully check the Login Template you plan to use for these samples. Check every field. Create a new template if you need to.

2.     Save the Heme-Quebec XLS file as tab-delimited text, in R:/hla/import. This import works for TXT only, not Excel format.

3.     Open HLA oracle, select Data Entry | Supplier Samples | Import Supplier File. Select the file, select supplier HEME-QUEBEC from the drop-down list of supplier names, give a name to the packing list, click Import.

4.     Select Data Entry | Supplier Samples | Login Packinglist, select the packing list. Preview and File records as normal

 

** Important: For the first import, select just 3-5 rows in the packing list to preview and file. Then do inquiry and check that all the data values are what you want. When it looks good, you can open this packinglist again and log in the rest of the rows.

 Version 3.24   Nov 14, 2009

         Allow two of the donor's antigens to match the same search antigen

Example: Search list is A2, A2, B15, B51

The matches for each item in this search list include the antigen itself and its CR matches:

A2 matches: A2 A28 A68 A69

A2 matches: A2 A28 A68 A69

B15 matches: B15 B62 B71 B72 B63 B50

B51 matches: B51 B52 B53 B35 B18 B78

 

The report (DEDHAM only) showed:

R# 066023 A2,A28,B51,B62 match

R# 275282 A*02, A*68, B*15, B*51 match

 

The report did NOT show:

R# 210836 A*02XX, A*68XX, B*51XX, B*53XX no match

 

Here's the reason for no match on R#210836:

- A*02XX matches the 1st search antigen A2 (direct)

- A*68XX matches the 2nd search antigen A2 (cross-react with A68)

- B*51XX matches the 4th search antigen B51 (direct)

- B*53XX also matches the 4th search antigen, but I exclude the 4th search antigen because it was already used to match the donor's B*51XX
