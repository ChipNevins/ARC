PROJECT STRATEGY - HLA ORACLE SYSTEM  8/15/02

 

 

 

  • Release 1: Final reports (opt 16/18), Patient family/URD reports, Lookup functions, Inquiry screen.  Bridge and conversion programs to support these functions.  Other functions - login, test ordering, result entry and exports - remain in Mumps. As data records are updated in Mumps, they are bridged across to Oracle (data is visible in Oracle within one minute after it is filed in Mumps).
  • Release 2: Result entry and NMDP transfer. Includes bridging LIFC and Reli results, Mumps option #6. Login remains in Mumps.
  • Release 3: Login

 

 

 

 

This project strategy converts the Mumps system in three releases,  starting at the back-end (results reports) and moving to the front-end (login).  Sharon and Chip selected this strategy because (1) we can deliver functionality earlier; (2) the users can actually use part of the system and offer important feedback sooner.  Also, there is a large benefit in implemeting the reports soon.

 

With staggered releases, there is a downside: we need to write bridge programs to keep the Oracle database in sync with the Mumps database.  But in our case this downside is not large, because the programs I use for the bridge are the same ones I need for the data conversion.  I will not need to write these programs twice.
