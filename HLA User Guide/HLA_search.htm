HLA Search

Last revision: April 3, 2007                         To print this page, Click here first

 

1)       HLA Antigen code table. The table now has a cross reactions field. For each antigen, you can enter a list of one or more cross-reacting antigens, separated by commas, e.g. A28, A32, A14.  You must define these in order to run the search report.

2)       HLA Search window parameters.

·         pA1, pA2, pB1, pB2 – two A values and two B values.  These are the patient/ recipient values. The system will check that all 4 values are in the Antigens table. Question: do you want to select a patient, or just enter the 4 values?

·         pMatch – Minimum number of matched values needed for a donor type to be returned as a match

·         pMismatch – Maximum number of miss-matched values. If more than this number is missed, the donor record will not be returned as a match.

·         pRuleOut – List of rule out antigens.  A donor with any of these will not be included in the report.

·         pDonorLocation – User can limit the search to donors in one location, or can select *ALL

·         pDonorAvailability – User can limit the search to only ACTIVE donors , or can select *ALL

·         pIgnoreCrossReactingAntigens – Enter Y if you do NOT want an antigen matched based on a cross-reacting value.  E.g, if A2 is cross-reactive with A28, a donor record will match on this antigen only if it is A2.

·         Donor R# range: Enter from and top values if you want to search only a range of r-numbers.  This would be used more in initial QA than in production.

3)       Search logic.

·         The donor’s first A antigen will be matched to either pA1 or pA2. If pA1 has already been matched, the match will be done on pA2

·         The donor antigens will be truncated to two digits for match, e.g. A*01XYZ will be truncated to A01 for the search

·         Only the A-locus and B-locus will be used in the search

·         A match can be a direct match, or a match based on a cross-reacting antigen unless the user answered ‘Y’ to pIgnoreCrossReactingAntigens

·         If a donor antigen is not a match, it is a mismatch; there is no other possibility.

4)       Presentation of search results.

·         As the system runs the search it will display number of records searched and number of records matched. The user can click a stop button to stop the search

·         When all records are searched, the system will display all data as a report, sorted by highest match to lowest match. You can print the report

·         The system automatically saves the report as a HTML file.  The message area at the bottom of the screen tells you the name of the file that was saved. You can open and edit this file using MS Word if you want to make any changes before sending out the report.

·         Note:  If you email the report as a Word file, the recipient can also change it. You may want to protect the file from edits before you send it.

5)       Elements of the report

·         Date report was run

·         Parameter values searched on

·         Patient name and R# (if user wants to lookup Pt to get match parameters)

·         Donor R#, location, HLA Type, and match/mismatch values for all matches

6)       Record of the search.  I do not plan to keep a record (in the database) of the searches run, but I could do this if it is needed.  We would also need a window to view these records.
