Setup of a new Employee to use Oracle HLA

  1. Oracle account. The Oracle DBA (Chip or Gil) creates an Oracle account for the new user. (This can be done remotely). The DBA adds these roles for each new account: CONNECT, RESOURCE, HLA_USER, LIFC_READER. If the user had a Mumps account, the DBA canmake the Oracle password the same as the Mumps password.
  2. EMPLOYEE Code table.
    • You must have security level 7 in Hla Oracle to perform this set of steps
    • In the HLA Oracle system, select Data Entry,Admin, Codes. the system opens the Codes maintenance window.
    • Select EMPLOYEE from the list of code types. Using the mouse, right-click in the list of Employe names, then select add.
    • Enter the Employee name, LAST,FIRST, employee initials
    • In the lower window, enter the ORACLE NAME and the SECURITY LEVEL. The ORACLE NAME value you enter must be the same as the account name which the DBA created. Normally this is the employee's last name.
    • These are the security levels: 1-View, 3-Login, 5-ResultEntry, 7-Dictionary maintenance.
    • Click SAVE. This employee can now use HLA Oracle
