Version 1.38, January 15, 2004 (Updated 2-Feb-04)

  • Bugfix build 2-Feb-04: Increased font size of results field in Batch/Single result reports, Fixed flgDelete problem in Contact Entry, so that new contacts show up in Patient Reports lookup list

  • Patient/Donor inquiry - corrected sort-order problem for people with more than 9 specimens.
  • Result reports, Batch and Single. New format which shows result date of each order.
  • Result Entry. Do not pre-assign preliminary readings to the HLA type if the specimen does not have any new HLA test orders. We had a few cases where a RELI result came over the bridge after the last update of HLA type, so when the user went into Result Entry for a new specimen, which had only an Antibody Screen order, the RELI reading was considered to be new, and it pre-filled. This problem is now corrected.
