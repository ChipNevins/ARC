Version 3.17  Aug 21, 2007

  • Dynal SBT reader is enhanced so that, if you redo the readings and import the file, it will create another set of preliminary readings, which you can select in Result Entry.  The reading list sorts so the most recent one is the first one listed for each locus.

Please try this first with one pc for 1 or 2 new reading files

  • NMDP Export lets you include or exclude loci DQA, DPA, DPB.  The exclusion is for the LBSW and the HML formats.  The TAB delimited format always includes all loci.

Version 3.16  Jul 8, 2007

·        Reader for Dynal SBT results.  Result files from r:\hla\sbtresults\ will be included in the HLA Reader (aka the Bridge). The files must be in one of these formats:   (1) Detail files with tab separators  (2)  Summary files with space separators.  I have attached examples of the class I and class II detail files, and a summary file, that Tatiana created for me to test the program at home.  Here is how to test this version:

Before loading 3-16

- Take the xls files out of the r:\hla\sbtreports folder first.  The reader does not understand them

- Run the reader to read any other import files

 Now load 3-16 on one PC

- Create a few detail or summary files, in the required format, put them in r:\hla\sbtreports\

- Run the reader

- Open the Inquiry Window for a R# that had results in the files, and verify that the prelim results are displayed

- Try Result Entry for one specimen

·         Memo fields. Changed bridge and Db so that patinet-level fields are kept with the person record, not in a separate table.

Version 1.11, release date 24-Jan-03

·         HLA Validation. A function to compare Mumps vs. Oracle HLA types. See further description on this site.

·         Specimens by Hospital report. A report of specimens received from a hospital. The user enters From date, To date, and Hospital as run time values.

Version 1.10, release date 14-Jan-03

·         Antigen List. This new window, under the Admin menu, displays the list of antigens. You can delete antigens so that they will not be accepted in Result Entry. You cannot add to the list from this window: new antigens will be added automatically when you use them in Result Entry

·         Validation. This new window compares the Oracle HLA types with the types in a text file which you have already generated from the Mumps system.

Version 1.9, release date 10-Jan-03

·         XML Reader. The reader now includes RELI files, and it keeps a log in R:\hla\Logs\

·         Inquiry. This function nows includes the reading results from RELI and Quicktype

Version 1.5, release date 08-Nov-02

·         Patient Reports. Changed method of bold/unbold - you can now use double-click. Fixed page numbering problem on printed copies.

·         Menu Selector. New menu item, Tools|Menu Selector, which opens a window witht he list of frequently used menu items. Single-click on an item to open that function. The menu selector stays open (in the background) for as long as you want it.

·         Final reports. Implemented **TEST VERSION** of final reports, for Sharon's review. A prominent header indicates that these are not production-mode reports.

·         Person Lookup window. Fixed problem - first user in was locking the recent list window, freezing others. Now fixed.

Version 1.4, release date 17-Oct-02

·         Patient Reports. You cannow establish a report for a patient who is not yet registered.

·         Dictionary Edit

·         Canned Comment entry

Version 1.3, release date 04-Oct-02

·         Patient Reports

·         Dictionary Edit

·         Canned Comment entry

Version 1.2, release date 23-Sep-02

·         XML Writer. Mumps program (not an Oracle program) which creates XML files for each new record or record update made in Mumps.

·         XML Reader. New window to read and file XML messages

·         Person Inquiry. New window to display and print patient/donor data.

Version 1.1, release date 09-Sep-02

·         SBT Window changes: Allow display of Inbox and Outbox files with single-click. Removes leading punctuation characters from the sequences when building them into output files. Displays a list of any outbox files which could not be interpreted and allows user to print the list.

Version 1.0, release date 16-Aug-02

·         LBSW filter. New window which lets the user convert Ref# to NMDP# in an LBSW-formatted file. Also allows change of Lab code.

·         SBT Window: New window which collects Sequence-based typing files into files grouped by locus. Output is a formatted NMDP SBT file.
